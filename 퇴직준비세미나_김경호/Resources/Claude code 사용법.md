# Claude Code 사용법

## 1. 설치

```bash
npm install -g @anthropic-ai/claude-code
```

- Node.js 18 이상 필요
- 설치 후 터미널에서 `claude` 명령어로 실행

## 2. 기본 사용법

```bash
# 대화형 모드로 시작
claude

# 질문을 직접 전달
claude "이 프로젝트의 구조를 설명해줘"

# 파이프로 입력 전달
cat error.log | claude "이 에러를 분석해줘"
```

## 3. 주요 슬래시 명령어

| 명령어 | 설명 |
|--------|------|
| `/help` | 도움말 표시 |
| `/clear` | 대화 초기화 |
| `/compact` | 대화 내용 요약하여 컨텍스트 절약 |
| `/cost` | 현재 세션의 토큰 사용량 및 비용 확인 |
| `/doctor` | Claude Code 설치 상태 점검 |
| `/init` | 프로젝트에 CLAUDE.md 생성 |
| `/memory` | CLAUDE.md 메모리 파일 편집 |
| `/review` | 코드 리뷰 요청 |
| `/vim` | Vim 모드 토글 |

## 4. 주요 기능

### 코드 작성 및 수정
- 파일 읽기, 쓰기, 편집 가능
- 기존 코드 패턴을 분석하여 일관된 스타일로 작성

### 터미널 명령 실행
- git, npm, docker 등 터미널 명령어 실행 가능
- 빌드, 테스트, 배포 작업 지원

### 코드베이스 탐색
- 프로젝트 구조 파악
- 파일 검색 및 코드 패턴 분석
- 버그 원인 추적

### Git 연동
- 커밋 메시지 작성
- PR 생성 및 리뷰
- 브랜치 관리

## 5. CLAUDE.md 파일

프로젝트 루트에 `CLAUDE.md` 파일을 만들면 Claude Code가 프로젝트 컨텍스트를 자동으로 인식합니다.

```markdown
# 프로젝트 설명
이 프로젝트는 ...

# 빌드 방법
npm run build

# 코딩 규칙
- TypeScript 사용
- 함수형 프로그래밍 선호
```

## 6. 유용한 팁

- **컨텍스트 관리**: 대화가 길어지면 `/compact`로 요약하여 성능 유지
- **구체적 지시**: "이 파일 수정해줘"보다 "src/app.ts의 login 함수에 에러 처리 추가해줘"가 효과적
- **단계적 작업**: 큰 작업은 작은 단위로 나누어 요청
- **확인 습관**: 코드 변경 후 빌드/테스트를 요청하여 검증
